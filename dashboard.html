Zero-Config Automatic Interception

NEON-SHIELD automatically intercepts traffic from all devices on your network without requiring any manual configuration on those devices:

  1. ARP Spoofing: This host announces itself as the network gateway via forged ARP replies, causing all target devices to route their traffic through this machine.
  2. Transparent Redirect: iptables rules silently redirect HTTP/HTTPS traffic (ports 80 & 443) to the proxy's transparent listeners.
  3. TLS Interception: The proxy intercepts HTTPS connections by generating certificates on-the-fly signed by a local Root CA. To see the content, devices must separately download and trust this CA (see below).
  4. Traffic Logging: All intercepted traffic is logged and visible in the loopback control panel: {CONTROL_PANEL_URL}
🔒 IMPORTANT: Unlike silent tools, devices connecting to untrustworthy Wi-Fi will still see TLS certificate errors in their browser, even if intercepted. This tool is NOT designed to bypass cert validation warnings — it's designed to show why you should never trust untrusted networks.
  1. Open Safari on your iOS device and browse to http://{LOCAL_IP}:{PORT}/ca.crt (or tap the download button above).
  2. A prompt will ask to download a configuration profile. Tap Allow.
  3. Open the Settings app on your device. You should see a Profile Downloaded button near the top. Tap it.
  4. Tap Install in the top-right corner, enter your passcode, and confirm the installation.
  5. Go back to the main Settings page, then navigate to: General → About → Certificate Trust Settings (at the very bottom).
  6. Locate EduLab Root CA under "Enable full trust for root certificates" and toggle the switch to ON.
  1. Open Chrome on your Android device and navigate to http://{LOCAL_IP}:{PORT}/ca.crt to download the certificate file.
  2. Open the device Settings and search for CA Certificate or Encryption & Credentials.
  3. Navigate to Install a certificate → CA certificate.
  4. You may see a warning indicating safety risks. Choose Install Anyway.
  5. Select the downloaded neon_shield_ca.crt file.
  6. Name the certificate (e.g., "Neon Shield CA") and tap OK/Install.

If you are routing traffic from another computer:

  1. Google Chrome / Microsoft Edge: Settings → Privacy and security → Security → Manage certificates → Trusted Root Certification Authorities → Import.
  2. Mozilla Firefox: Settings → Privacy & Security → Certificates → View Certificates → Authorities → Import. Mark "Trust this CA to identify websites".
  3. Linux (Ubuntu/Debian):
